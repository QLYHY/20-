# 第20届全国大学生智能汽车竞赛 - 平衡轮腿组 国家二等奖代码开源

## 项目简介

本项目为第20届全国大学生智能汽车竞赛（NXP智能车竞赛）**平衡轮腿组**参赛代码，最终获得**全国二等奖（国二）**。

> 项目基于逐飞科技（SeekFree）TC264开源库开发，遵循 GPL 3.0 开源协议。

## 硬件平台

| 组件 | 型号 | 说明 |
|------|------|------|
| 主控芯片 | **Infineon AURIX TC264D** | 双核 Tricore 架构，200MHz |
| 开发环境 | **AURIX Development Studio (ADS)** | v1.9.20 |
| 底层库 | **逐飞科技 TC264 开源库** | V3.3.1 |
| 摄像头 | MT9V03X | 灰度摄像头，188×120 |
| 惯性测量单元 | IMU660RA | 6轴姿态传感器 (SPI) |
| ToF 测距 | DL1B | 激光测距模块 (100Hz) |
| 显示屏 | IPS200 | 2.0寸 TFT，240×320 (SPI) |
| 电机驱动 | 无刷电机 + 舵机 | 平衡 + 转向 + 腿部驱动 |
| 无线通信 | 无线串口模块 | 调试与参数调节 |

## 机械结构

- **腿部机构**：并联五连杆机构，4个舵机（L1, L2, R1, R2）驱动
- **腿部参数**：leg1=60mm, leg2=90mm, leg3=90mm, leg4=60mm, L5=35mm
- **平衡控制**：2个无刷电机差速驱动，基于轮腿倒立摆模型
- **运动方式**：轮式平衡行驶 + 腿部高度调节

## 控制算法

### 姿态解算
- **四元数姿态更新**：基于 IMU660RA 的 6 轴数据
- **Mahony 互补滤波**：融合加速度计与陀螺仪数据，Kp=0.06, Ki=0.004
- **5ms 更新周期**

### 平衡控制（串级 PID）

```
期望速度 → [速度环 PID] → [角度环 PID] → [角速度环 PID] → PWM输出
                ↑                    ↑              ↑
            编码器反馈          姿态角反馈      陀螺仪角速度反馈
```

| 控制环 | 说明 | 核心参数 |
|--------|------|----------|
| 速度环 | 位置式PID，控制行驶速度 | Kp=0.04, Ki=-0.045, Kd=-0.018 |
| 角度环 | 位置式PID，控制车身倾角 | Kp=45 |
| 角速度环 | 增量式PID，稳定车身姿态 | Kp=0.65, Ki=0.038 |

### 转向控制
- **视觉偏差量输入**：基于赛道边界加权平均误差
- **PD控制**：Kp=23, Kd=0, 带GKD角速度前馈
- **二次项补偿**：steering_KP2=0.8 (误差非线补偿)

### 腿部运动学
- **并联五杆机构逆运动学解算**：根据目标足端坐标 (X, Y) 求解关节角度 (α, β)
- **腿部高度控制**：Y_demand 控制车身高度，Kp_Y=1
- **水平位置控制**：X 由速度环PWM控制，Kp_X=1
- **腿部限幅**：30mm ≤ Y ≤ 120mm，X范围受机械限位

### Roll 稳定控制
- 单边桥通过时的横滚角闭环控制
- roll_angle_KP=-60, roll_angle_KD=50

## 赛道元素识别

### 图像处理流程
1. **大津法 (OTSU) 二值化**：动态阈值分割
2. **形态学滤波**：3×3窗口腐蚀+膨胀
3. **最长白列巡线**：寻找左右边界
4. **八邻域边界跟踪**：精确提取赛道边缘

### 识别的赛道元素

| 元素 | 识别方法 | 状态机 |
|------|----------|--------|
| **环岛** | 单调性突变 + 连续性撕裂 + 四拐点检测 | 6状态机 (进下口→进上口→环内→出下口→过上口→退出) |
| **十字路口** | 四拐点检测 + 双边丢线 + 斜率补线 | 拐点连线补线策略 |
| **单边桥** | 边界突变检测 (≥15像素) + 双边确认 | 识别→确认→通过→计数退出 |
| **避障** | 同上 + 赛宽突变 | 识别→绕障→恢复 |
| **斑马线** | 边界内黑白跳变统计 (≥6行) | 识别后减速停车 |
| **坡道** | ToF DL1B 测距 (≤30cm) + 赛宽 | 识别→准备→上坡→完成 |
| **跳跃** | 双边界同时丢失 | 蓄力→跳跃→着陆恢复 |

### 元素执行顺序
- 基于 `Element_Order[]` 数组预置元素顺序
- Flash 存储元素排列，开机自动读取

## 任务调度

基于 PIT (CCU6) 定时器中断实现多任务调度：

| 定时器 | 周期 | 任务 |
|--------|------|------|
| CCU60_CH0 | 5ms | 姿态解算 + 按键扫描 |
| CCU60_CH1 | 1ms | 核心控制（平衡环、转向环、速度环）|
| CCU61_CH0 | 5ms | 预留 |
| CCU61_CH1 | 1ms | 斑马线里程累计、ToF数据处理 |

### 核心控制 1ms 调度

```
1ms:   角速度环 + 逆运动学解算 + 舵机输出
5ms:   转向计算 + 角度环
10ms:  ToF 测距
15ms:  Roll 平衡控制
20ms:  速度环 + 编码器测速
```

## 双核架构

- **CPU0 (主核)**：系统初始化、菜单显示、参数调节、启动控制
- **CPU1 (辅核)**：图像采集与处理、赛道元素识别

两核通过 `cpu_wait_event_ready()` 同步启动。

## IO 分配

详见 `推荐IO分配.txt`：
- 摄像头：8位数据口 + VSYNC(P02_0) + PCLK(P02_1) + UART(P02_2/P02_3)
- 舵机：4路 PWM (P21_2/P21_3/P21_4/P21_5)
- 电机：4路 PWM (P02_4/P02_5/P02_6/P02_7)
- 编码器：4路 (含LSB/DIR)
- IMU：SPI0 (P20_11~P20_14)
- 屏幕：SPI2 (P15_0~P15_5)
- 蜂鸣器：P33_9
